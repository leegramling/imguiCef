# Windows Icon Setup for ImGuiCefVulkan

This project uses embedded Windows resources for proper taskbar icon display on Windows 11. This guide covers both the Python generation approach and the Visual Studio 2022 approach.

## Files Added

### Embedded Resource Implementation
- `resources/app.rc` - Windows resource file with embedded icon
- `resources/resource.h` - Resource definitions
- `resources/browser.ico` - Rocket-themed icon file (embedded in executable)
- `create_icons.py` - Python script to generate the rocket icon
- Updated `CMakeLists.txt` to compile resources on Windows builds

## Creating Icons with Visual Studio 2022

### Method 1: Using Visual Studio 2022 Built-in Tools

1. **Open your project in Visual Studio 2022**

2. **Add Resource Files to Project**:
   - Right-click your project in Solution Explorer
   - Select **Add → New Item**
   - Choose **Visual C++ → Resource → Resource File (.rc)**
   - Name it `app.rc` and place it in a `resources` folder

3. **Create the Icon Resource**:
   - Right-click the new `app.rc` file in Solution Explorer
   - Select **Open With → Resource View**
   - In Resource View, right-click and select **Add Resource**
   - Select **Icon** and click **New**
   - This creates a default icon with ID `IDI_ICON1`

4. **Edit the Icon**:
   - Double-click the icon resource to open the icon editor
   - Use Visual Studio's built-in icon editor to draw your icon
   - **Important**: Create multiple sizes (16x16, 32x32, 48x48)
   - Right-click in the icon editor and select **New Image Type**
   - Add 16x16, 32x32, and 48x48 versions

5. **Resource Header File**:
   - Visual Studio automatically creates `resource.h` with the icon ID
   - The ID will be something like `#define IDI_ICON1 101`

### Method 2: Import Existing ICO File

1. **Generate ICO with Python** (if you want the rocket design):
   ```bash
   python3 create_icons.py
   ```

2. **Add Resource File**:
   - Right-click project → **Add → New Item → Resource File (.rc)**
   - Name it `app.rc` in `resources` folder

3. **Import the ICO File**:
   - Right-click `app.rc` → **Open With → Resource View**
   - Right-click in Resource View → **Add Resource → Import**
   - Browse to `icons/browser.ico` and import it
   - Visual Studio will assign it an ID like `IDI_ICON1`

4. **Verify Resource Content**:
   Your `app.rc` should contain:
   ```rc
   #include "resource.h"
   IDI_ICON1 ICON "browser.ico"
   ```

   Your `resource.h` should contain:
   ```c
   #define IDI_ICON1 101
   ```

## How It Works

### Embedded Resource Approach

Windows automatically uses embedded icon resources for:
- **Taskbar icons** - What appears in the Windows taskbar
- **Title bar icons** - Small icon in window title bar
- **Alt+Tab switcher** - Icon shown when switching between applications
- **Task Manager** - Icon displayed in process list
- **File associations** - Icon for the executable file

**How it works:**
- Icon is compiled directly into the `.exe` file during build
- Windows automatically detects and uses `IDI_ICON1` resource
- No runtime loading required - works immediately
- Guaranteed to work on all Windows versions
- No external files needed - completely self-contained

### Rocket Icon Design

The embedded icon features:
- **Rocket Shape**: Classic rocket with pointed nose cone and stabilizer fins
- **Fire Trail**: Multi-layered exhaust with orange-to-yellow gradient
- **15° Right Tilt**: Dynamic angled appearance
- **Multi-Size Support**: 16x16, 32x32, and 48x48 sizes in single ICO file
- **Progressive Detail**: More details visible at larger sizes

## Integration

No code integration required! The icon is automatically embedded during Windows builds:

1. **CMake detects Windows** and includes `resources/app.rc`
2. **Resource compiler** embeds the icon into the executable
3. **Windows automatically** uses the embedded icon for taskbar/title bar

## CMake Integration (Works with Visual Studio)

For CMake-based projects (like this one), which also makes resource files appear in Visual Studio:

```cmake
# Windows resource files for embedded icon
if(WIN32)
    set(RESOURCE_FILES
        "${CMAKE_CURRENT_SOURCE_DIR}/resources/app.rc"
        "${CMAKE_CURRENT_SOURCE_DIR}/resources/resource.h"
        "${CMAKE_CURRENT_SOURCE_DIR}/resources/browser.ico"
    )

    # Add resource files to project for Visual Studio
    foreach(RESOURCE_FILE ${RESOURCE_FILES})
        if(EXISTS "${RESOURCE_FILE}")
            list(APPEND SOURCES ${RESOURCE_FILE})
            message(STATUS "Added resource file: ${RESOURCE_FILE}")
        else()
            message(WARNING "Resource file not found: ${RESOURCE_FILE}")
        endif()
    endforeach()

    # Set source groups for Visual Studio organization
    source_group("Resource Files" FILES ${RESOURCE_FILES})
endif()
```

This CMake configuration:
- ✅ Includes all resource files in the build
- ✅ Makes them visible in Visual Studio Solution Explorer
- ✅ Organizes them in a "Resource Files" folder
- ✅ Provides helpful build messages

## Troubleshooting Visual Studio 2022

### Common Issues and Solutions

1. **"Cannot find resource.h"**:
   - Make sure `resource.h` is in your project and included in the .rc file
   - Check that the path in the `#include "resource.h"` statement is correct
   - Verify resource.h contains the icon definition: `#define IDI_ICON1 101`

2. **"Cannot find icon file"**:
   - Verify the icon file path in the .rc file is correct
   - Paths in .rc files are relative to the .rc file location
   - Example: If .rc is in `resources/` and icon is in `resources/`, use just `"browser.ico"`

3. **Resource View not showing**:
   - Go to **View → Other Windows → Resource View**
   - If Resource View is empty, right-click and **Add Resource**

4. **Icon not updating after changes**:
   - Clean and rebuild the solution (**Build → Clean Solution**, then **Build → Rebuild Solution**)
   - Sometimes Visual Studio caches the old icon

5. **Wrong icon ID**:
   - Ensure your icon has ID `IDI_ICON1` with value `101`
   - This should be the first (default) icon resource in your application
   - Windows automatically uses the first icon resource for the application
